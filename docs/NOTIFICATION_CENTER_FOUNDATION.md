# Notification Center Foundation

M35 upgrades KasetHub notifications into a unified local/mock notification center.

## Current Behavior

- `/app/notifications` aggregates local/demo items from weather alerts, Crop Watch, AI credits, community moderation, My Farm next actions, content fixtures, Guest Sync readiness, and system notices.
- `/app/notification-settings` stores local mock preferences in `kasethub.notificationCenter.v1`.
- Read/unread and "clear read demo" state are localStorage-only.
- Summary cards link to the notification center from home, profile, My Farm, and admin.

## Notification Types

- `weather_alert`
- `crop_price_alert`
- `ai_credit`
- `community_report`
- `moderation_update`
- `my_farm_reminder`
- `content_update`
- `account_sync`
- `system_notice`

## Local Data Boundary

M35 does not request push permission, register a service worker, send LINE messages, send SMS/email, call a backend, write Supabase data, or run a scheduler. All items are local/mock/demo and should be labeled as such in the UI.

## Preference Rules

Preferences are in-app display controls only. They are not delivery consent for production. Future production delivery must collect explicit consent per channel, support quiet hours, allow unsubscribe, and record delivery/audit logs server-side.

## Source Rules

Every notification should keep a visible source label and demo/local label. Price notifications must keep `ราคาอ้างอิง` language. Weather notifications must not claim a real forecast. Moderation notifications must state that there is no real admin action in the current version.

## Future Backend Mapping

Future backend tables can include:

- `notification_preferences`
- `notification_events`
- `notification_deliveries`
- `notification_digest_jobs`

Production events should be generated by backend jobs/functions from trusted source records, not by browser-only state.
