# Farm Area Measurement Foundation

M33 adds a local/mock farm area measurement foundation for KasetHub. It helps farmers estimate plot size, convert Thai land units, save local plot estimates, and understand how future GPS/map measurement could work.

This is not an official land survey feature.

## Current Boundary

- No real GPS.
- No map API.
- No geolocation permission request.
- No network call.
- No backend write.
- No Supabase write.
- No production land survey claim.
- Saved plot records stay in localStorage only.

Required user-facing disclaimer:

> เป็นการคำนวณประมาณการ ไม่ใช่การรังวัดที่ดินอย่างเป็นทางการ

## Local Calculator

Supported shapes:

- `rectangle`: width x length
- `square`: side x side
- `triangle`: base x height / 2
- `custom_polygon_mock`: manually entered estimated square meters for future polygon planning

All entered dimensions are treated as meters. Invalid or zero values return warnings instead of throwing errors.

## Thai Land Unit Rules

- 1 ตารางวา = 4 ตารางเมตร
- 1 งาน = 100 ตารางวา = 400 ตารางเมตร
- 1 ไร่ = 4 งาน = 1,600 ตารางเมตร

Additional conversion units:

- hectare = 10,000 square meters
- acre = 4,046.8564224 square meters

## Local Storage

M33 uses:

- `kasethub.farmArea.v1`

Stored data includes:

- plot id and local id
- plot name
- shape
- method
- accuracy level
- calculated area labels
- input dimensions
- created timestamp and display label
- disclaimer

Raw map geometry, GPS tracks, coordinates, files, and server identifiers are not stored.

## User Experience

`/app/farm-area` shows:

- simple shape selector
- large readable number inputs
- calculated area in square meters, ตารางวา, งาน, ไร่, hectare, and acre
- local-only save button
- saved plot cards with remove action
- Thai unit rules
- future GPS/map planning panel

`/app/farm-area-guide` explains:

- manual tape measurement
- rectangle, square, and triangle estimation
- Thai land units
- accuracy cautions
- why official land matters still require official survey documents

## My Farm Integration Future

M33 only linked the area planner from My Farm. M34 reads saved local plot estimates into the My Farm Hub dashboard and timeline, but it still does not merge plot estimates into production My Farm records.

Future integration can let authenticated users attach plot measurements to:

- farms
- crop seasons
- plant analysis history
- irrigation planning
- weather/price alerts by farm area
- crop cost estimates

## Backend/Supabase Future Schema

Future Supabase planning should include:

- `farm_plots`
- `farm_plot_measurements`
- `farm_plot_boundaries`
- `farm_plot_notes`

Recommended boundaries:

- User-owned plot records require Supabase Auth.
- RLS must limit reads/writes to the plot owner.
- Precise boundary coordinates should be private by default.
- Official survey claims should never be generated by the app.
- Export/share requires explicit user confirmation.

## Production Blockers

- Real auth and RLS are required before cloud plot storage.
- Location privacy review is required before GPS/geolocation.
- Map provider terms and costs must be reviewed before map API use.
- Boundary accuracy must be clearly labeled.
- Survey/legal copy must be reviewed before any official-looking land document feature.
