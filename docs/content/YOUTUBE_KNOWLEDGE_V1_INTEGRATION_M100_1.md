# YouTube Knowledge V1 Integration M100.1

## 1. Purpose

The owner has an agriculture YouTube channel with about 36k followers. V1 should use this audience as the first real feedback loop for KasetHub.

The app should not overbuild CMS or YouTube automation before V1. It should use simple content and audience feedback to shape the AI-first utility experience.

## 2. Audience Feedback Plan

Use YouTube videos, descriptions, community posts, or pinned comments to ask:

- What do you want to ask AI about farming?
- Which crop problems do you see often?
- What weather warnings help you decide farm work?
- Which calculators do you actually use?
- Would you keep a simple farm notebook in the app?

Collect patterns manually before building complex backend feedback systems.

## 3. App CTA Ideas For Later

Future video descriptions can include:

- `ลองถาม AI เกษตรใน KasetHub`
- `เช็กอากาศและความเสี่ยงเกษตร`
- `ใช้เครื่องมือคำนวณปุ๋ย ระยะปลูก และต้นทุน`
- `บันทึกข้อมูลฟาร์มง่าย ๆ ในเครื่องของคุณ`

Do not promise official diagnosis, guaranteed results, or expert replacement.

## 4. Starter Content Focus

Useful starter topics:

- soil basics
- fertilizer basics
- weather risk
- pest and disease basics
- land/area tools
- planting distance
- farm records basics
- cost and profit basics

Content should be short, Thai-first, and connected to actions in the app.

## 5. V1 Knowledge Area Direction

Keep articles/videos simple:

- show practical starter content
- link to AI for follow-up questions later
- link to calculators when relevant
- avoid heavy CMS workflows before V1
- avoid YouTube API expansion unless separately scoped

## 6. Safety Boundary

Content and AI should avoid:

- unsafe pesticide rates
- dangerous chemical mixing
- guaranteed disease diagnosis
- guaranteed yield/profit claims
- legal, tax, loan, or health decisions

When content discusses high-risk actions, it should point users to labels, local officers, experts, and trusted sources.

## 7. No M100.1 Feature Expansion

M100.1 does not add YouTube API integration, backend feedback submission, notifications, cloud sync, or new CMS systems.
